状态码

100 请求中
200 响应成功

301 永久重定向
302 暂时重定向（网址URL劫持）
304 未修改 IfModifiedSince

400 参数错误 请求参数错误
401 未授权 身份验证不通过
403 禁止访问 服务器拒绝请求
404 未找到 服务器找不到请求的网页
405 请求方法错误
415 不支持的媒体类型

500 服务器异常
502 错误网关 服务器作为网关或代理，从上游服务器收到无效响应
504 网关超时 服务器作为网关或代理，没有及时从上游服务器收到请求

TCP三次握手四次挥手

三次握手

  1. 客户端发送SYNbit=1, Seq=x序列号给服务端
  2. 服务端发送SYNbit=1, Seq=y序列号,ACKbit = 1,ACKnum = x+1确认码给客户端
  3. 客户端发送,ACKbit = 1,ACKnum = y+1确认码给服务端

四次挥手

  1. 客户端发送FINbit = 1, seq = x
  2. 服务端发送ACKbit = 1, ACKnum = x +1
  3. 服务端发送Finbit = 1,seq = y
  4. 客户端发送ACKbit = 1,ACKnum = y + 1

客户端等待固定时间，等待可能出现的要求重传的ACK包。如果没有收到服务端的ACK，认为服务端已经正常关闭连接，于是自己也关闭连接。（两个最大阶段生命周期 2MSL，2 Maximum Segment Lifetime）。
RFC 793中规定MSL为2分钟，实际应用中常用的是30秒，1分钟和2分钟等。

  5. 浏览器解析渲染页面
